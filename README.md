## 提交规范
### Type
- `feat` 新功能 feature
- `fix`  修复 bug
- `docs`  文档注释
- `style`  代码格式(不影响代码运行的变动)
- `refactor`  重构、优化(既不增加新功能，也不是修复bug)
- `perf`  性能优化
- `test` 增加测试
- `chore`  构建过程或辅助工具的变动
- `revert` 回退
- `build`  打包
- `close` 关闭 issue

### scope 选填
`commit` 的作用范围，可以是模块，也可以是模块内的视图层，模型层或者其他

### subject
`commit` 的描述

> Type(scope): subject

例如修复 `Permissions` 模块的 bug: **fix(Permissions): 用户列表查询错误**
